# 1. 개발 목표 및 가이드 (Development Goals & Guidelines)

## 🎯 서비스의 궁극적 목표 (Core Vision)
RideKorea는 단순한 자전거 속도/경로 추적 앱(예: Strava)이 아닙니다. **"대한민국을 방문한 외국인 자전거 여행객(인바운드)이 길 잃을 두려움 없이 로컬 문화를 탐험하고, 그들의 모든 여정을 한 편의 감성적인 기행문으로 완성시켜 주는 소셜 여행 플랫폼"**입니다.

## 🚀 3대 핵심 개발 철학
1. **기록이 아닌 '기행문(Diary)'**: 사용자의 위치, 속도 등 차가운 데이터뿐만 아니라 사진, 감정, 머무른 시간 등 따뜻한 맥락(Context)을 함께 묶어 저장해야 합니다.
2. **실수(경로 이탈)의 긍정적 치환**: 길을 잘못 들었다고 경고음을 울리는 대신, 원래 경로(Blue)와 이탈한 경로(Pink)를 시각적으로 분리하여 "나만의 새로운 개척로"로 포장해 주는 UX를 지향합니다.
3. **지역 경제(로컬)와의 상생**: 지자체의 지원을 이끌어내기 위해, 특정 지역 진입 시 GPS 기반으로 실시간 지역 상권 바우처(쿠폰)를 팝업으로 띄우는 기능이 최우선 비즈니스 모델(BM)로 작동해야 합니다.

## 💻 AI 협업 개발 가이드라인 (For GPT / Claude)
이 문서를 바탕으로 코드를 생성할 AI 에이전트들은 다음 원칙을 반드시 준수해야 합니다.
* **Component-Driven**: 모든 UI 요소는 재사용 가능한 작은 단위의 컴포넌트(Component)로 분리하여 설계하십시오.
* **Strict Typing**: TypeScript를 사용하여 프론트엔드의 모든 Props와 State, Supabase DB 스키마에 대한 엄격한 타입(Type)을 정의하십시오.
* **Offline-Resilience**: 자전거 여행 중에는 네트워크 음영 지역(산간, 터널)을 지나게 됩니다. GPS 좌표와 사진은 기기(Local Storage/SQLite)에 우선 캐싱되고, 네트워크가 복구될 때 Supabase 서버로 백그라운드 동기화되도록 아키텍처를 짜야 합니다.
