# RideKorea POI Data Sources and License Register

작성일: 2026-06-30

## 목적

RideKorea의 수리소, 식당, 숙소, 교통, 문화, 경치 POI 데이터를 실제 운영 데이터로 확장하기 전에 출처, 이용허락, 갱신 주기, 표시 의무를 한 곳에서 관리한다.

이 문서는 개발자가 임포터를 실행하기 전에 확인해야 하는 운영 기준이며, 실제 데이터셋을 추가할 때마다 아래 레지스터를 갱신한다.

## 공식 기준 요약

공공데이터포털 정책 기준으로, 포털에서 제공 중인 공공데이터는 별도 신청 없이 이용할 수 있다. 다만 제공되지 않는 데이터는 제공신청이 필요할 수 있고, 공공데이터법상 제외 대상 정보가 포함되면 제공이 거부될 수 있다.

저작권 등 제3자 권리가 포함된 데이터는 권리자의 이용허락 범위를 확인해야 하며, 저작물이 포함된 공공데이터는 공공누리 유형을 확인해야 한다.

RideKorea는 상업 서비스 가능성을 전제로 하므로, 라이선스 판정은 아래처럼 보수적으로 한다.

| 구분 | RideKorea 판정 | 운영 규칙 |
| --- | --- | --- |
| 공공누리 제0유형 | 즉시 사용 가능 | 출처표시 조건은 없지만 운영 추적을 위해 출처와 수집일을 기록한다. |
| 공공누리 제1유형 | 사용 가능 | 앱 POI 상세 또는 데이터 고지 화면에 출처표시를 준비한다. |
| 공공누리 제2유형 | 보류 | 비상업적 이용만 가능하므로 상업 서비스에는 기본적으로 쓰지 않는다. |
| 공공누리 제3유형 | 검토 필요 | 변경 금지 조건이 있으므로 번역, 정제, 요약, 좌표 보정이 가능한지 법무 검토한다. |
| 공공누리 제4유형 | 보류 | 비상업 + 변경 금지 조건이라 운영 데이터로 쓰지 않는다. |
| AI 학습용 유형 | POI 운영 데이터로는 별도 검토 | 학습 목적과 앱 표시 목적이 다를 수 있으므로 데이터별 고지를 확인한다. |
| 민간/지도/플랫폼 약관 | 계약 또는 명시 허락 필요 | 화면 표시, 저장, 재배포, 파생 데이터 생성을 각각 확인한다. |
| 사용자 제보/리뷰 | 서비스 약관과 동의 필요 | 작성자 권리, 공개 범위, 신고/삭제 절차, 개인정보 포함 여부를 관리한다. |

## 데이터 도입 원칙

1. 출처 URL, 데이터명, 제공기관, 라이선스, 수집일을 모르면 임포트하지 않는다.
2. 상업적 이용 가능 여부가 불명확하면 임포트하지 않는다.
3. 번역, 요약, 좌표 보정, 카테고리 매핑이 금지될 수 있는 데이터는 임포트하지 않는다.
4. 개인 전화번호, 개인 주소, 차량번호, 얼굴 식별 사진 등 개인정보 가능성이 있으면 임포트하지 않는다.
5. 지도 제공사의 장소 검색 결과를 우리 DB에 장기 저장하려면 해당 지도 제공사의 약관 또는 계약을 별도로 확인한다.
6. 출처표시가 필요한 데이터는 POI 상세 화면과 데이터 고지 화면에서 표시할 수 있도록 attribution 값을 남긴다.

| 우선순위 | 출처 후보 | 적용 카테고리 | 상태 | 라이선스 확인 | 비고 |
| --- | --- | --- | --- | --- | --- |
| 1 | 공공데이터포털(data.go.kr) 개별 데이터셋 | repair, lodging, culture, transport | 후보 | 데이터셋별 공공누리 유형 확인 필요 | 포털 정책만으로 충분하지 않고 개별 데이터의 이용허락을 확인해야 한다. |
| 1 | 지자체 열린데이터 광장 | food, lodging, culture, scenic | 후보 | 페이지별 라이선스 확인 필요 | 서울/부산/인천/전북/전남/제주 등 구간별로 우선 검토한다. |

| 2 | 제휴 업체 직접 제공 데이터 | repair, food, lodging, voucher | 계약 필요 | 제휴 계약서 필요 | 바우처, 추천 노출, 가격 정보는 계약 기반으로만 운영한다. |
| 3 | 사용자 제보 POI | repair, food, lodging, scenic | 약관 필요 | 사용자 콘텐츠 약관 필요 | 신고, 숨김, 운영자 검수, 작성자 표시 정책이 먼저 필요하다. |

## 임포트 전 필수 메타데이터

현재 CSV/JSON 임포터는 POI 본문 필드를 처리한다. 실제 운영 데이터는 아래 메타데이터를 별도 관리 문서나 sidecar CSV로 함께 보관한다. 다음 스키마 확장 때 `travel_pois`에 편입한다.

| 필드 | 필수 | 설명 |
| --- | --- | --- |
| `source_url` | 필수 | 원천 데이터셋 또는 공식 안내 페이지 URL |
| `source_name` | 필수 | 데이터셋 이름 또는 제공기관 이름 |
| `license_type` | 필수 | 예: KOGL-0, KOGL-1, partner-contract, user-generated |
| `commercial_use_allowed` | 필수 | 상업 서비스 사용 가능 여부 |
| `derivative_allowed` | 필수 | 번역, 요약, 좌표 보정, 카테고리 매핑 가능 여부 |
| `attribution` | 조건부 필수 | 출처표시 문구 |
| `retrieved_at` | 필수 | 수집 또는 확인일 |
| `reviewer` | 필수 | 라이선스를 확인한 운영자 |
| `review_status` | 필수 | approved, needs-review, rejected |

## 임포트 승인 체크리스트

- 데이터셋 URL과 제공기관을 확인했다.
- 라이선스 또는 약관을 확인했다.
- 상업적 이용 가능 여부를 확인했다.
- 번역, 정제, 좌표 보정이 가능한지 확인했다.
- 출처표시 문구를 확정했다.
- 개인정보나 민감정보가 포함되지 않았는지 확인했다.
- `source + external_id`가 안정적인 중복 기준인지 확인했다.
- 샘플 CSV/JSON이 `--dry-run`을 통과했다.
- 운영자가 데이터 품질과 좌표 정확도를 샘플링 검수했다.

## 앱 표시 정책

- POI 상세에는 데이터 출처와 마지막 확인일을 표시할 수 있어야 한다.
- 출처표시가 필요한 데이터는 화면 하단이나 상세 정보 영역에 숨기지 않고 표시한다.
- 교통 규정은 "마지막 확인일"을 반드시 보여준다.
- 폐업, 오류, 위험 제보가 누적된 POI는 운영자 검수 전까지 지도에서 숨길 수 있어야 한다.

## 사용자가 준비해주면 좋은 것

실제 데이터를 들여올 때는 아래 정보가 있으면 바로 임포트 후보로 검토할 수 있다.

- 데이터셋 또는 공식 안내 페이지 URL
- 제공기관명
- 공공누리 유형 또는 이용약관 링크
- 출처표시 문구
- 갱신 주기
- 우선 적용할 구간: 예) 인천공항-아라서해갑문, 서울-부산, 제주 환상자전거길
