# RideKorea 디자인 전면 개편 계획서

작성일: 2026-06-29  
목적: 기존 디자인 가이드와 현재 앱 구현 상태를 바탕으로, RideKorea를 외국인 라이더 대상 상용 MVP 수준의 모바일 앱으로 재설계한다.

---

## 1. 현재 디자인 진단

### 1.1 문서상 디자인 방향

`docs/ridekorea_design_guideline.md`에는 다음 방향이 이미 정의되어 있다.

- 지도 중심 UX: The Map is the Hero
- 라이딩 중 빠르게 읽을 수 있는 Glanceable UX
- 정차 후 조작을 전제로 한 Safe-Stop Interaction
- K-Indigo, Aero Blue, Asphalt Gray 기반 색상 체계
- Journey / Moments / My Path / Compass 4탭 구조
- 네이버 지도 중심의 지도 레이어 UX

### 1.2 현재 코드와의 차이

현재 구현은 디자인 가이드와 아직 연결되어 있지 않다.

- `frontend/src/constants/theme.ts`는 Expo 기본 흑백 테마에 가깝다.
- `frontend/src/app/_layout.tsx`는 `Home`, `Explore` 2탭 구조다.
- 핵심 기능이 `frontend/src/app/index.tsx` 한 파일에 몰려 있어 화면별 경험 설계가 어렵다.
- 지도, 모달, 어드민, 바우처, 다이어리 UI가 한 화면에 섞여 있어 사용자 흐름이 무겁다.
- 디자인 토큰, 버튼, 패널, 리스트, 바텀시트 같은 공통 컴포넌트 체계가 없다.

## 2. 디자인 개편 원칙

### 2.1 첫 화면은 설명이 아니라 로그인

RideKorea는 개인 주행 기록, 사진 일지, 바우처 지갑을 다루므로 앱 진입 시 인증 상태를 먼저 확인한다.

- 비로그인 사용자의 첫 화면은 SNS 로그인 화면이다.
- 1차 개발 범위에서는 Google 로그인만 제공한다.
- Apple 로그인은 iOS 출시 준비 단계에서 추가한다.
- 로그인 성공 후 첫 화면은 Journey 지도다.
- 마케팅성 랜딩 화면은 만들지 않는다.
- 로그인 화면은 앱 설명을 길게 늘어놓지 않고, RideKorea의 목적과 Google 로그인 액션만 명확하게 보여준다.
- 지도 위 UI는 로그인 이후 Journey 화면에서 필요한 정보만 띄운다.
- 상세 정보는 하단 시트와 탭 화면으로 이동시킨다.

### 2.2 라이더가 멈춰서 조작하는 UI

주행 중 조작을 유도하지 않는다.

- 주요 버튼은 크고 간결하게 만든다.
- 작은 텍스트 버튼을 남발하지 않는다.
- 숫자, 거리, 다음 인증센터, 남은 거리처럼 순간적으로 읽어야 하는 정보는 크게 보여준다.
- 설정/필터/작성 같은 복잡한 작업은 정차 후 가능한 화면으로 분리한다.

### 2.3 외국인 친화 정보 구조

외국인에게 낯선 한국 자전거 종주 문화를 앱 안에서 자연스럽게 설명한다.

- 코스명, 인증센터명, 주소, 설명은 영어 우선 노출한다.
- 한국어명은 보조 정보로 병기한다.
- 인증센터, 스탬프, 자전거 휴대 대중교통, 숙박/캠핑 정보는 별도 정보 카드로 정리한다.
- 지도 위 POI는 너무 많이 보여주지 않고, 라이더에게 필요한 것만 레이어로 제어한다.

## 3. 새 IA 및 탭 구조

### 3.0 Auth Gate

목적: 인증 상태에 따라 로그인 화면 또는 앱 본 화면으로 보낸다.

핵심 UI:

- RideKorea 로고/서비스명
- 짧은 한 줄 가치 제안
- Continue with Google 버튼
- Apple 로그인 준비 상태 표시 없음
- 약관/개인정보 처리방침 링크
- 로그인 진행/실패 상태

동작:

- 저장된 JWT가 유효하면 Journey로 이동한다.
- 토큰이 없거나 만료되면 Login 화면을 보여준다.
- Google 로그인 성공 후 백엔드 JWT를 SecureStore에 저장한다.

### 3.1 Journey

목적: 지도와 현재 라이딩의 중심 화면.

핵심 UI:

- 전체 화면 네이버 지도
- 현재 위치 버튼
- 코스 선택 칩
- 레이어 버튼
- 하단 라이딩 패널
- 선택한 스팟 상세 바텀시트
- Start / Pause / Finish riding 액션

주요 정보:

- 현재 코스
- 다음 인증센터
- 남은 거리
- 공개 다이어리 마커
- 바우처 활성 스팟

### 3.2 Moments

목적: 공개 다이어리와 실제 라이더 경험을 보는 커뮤니티 피드.

핵심 UI:

- 사진 중심 피드 리스트
- 스팟명, 코스명, 날짜
- 작성자 프로필
- 미니 지도 또는 위치 태그
- 신고/공개 상태 제어

### 3.3 My Path

목적: 나의 종주 기록, 다이어리, 바우처를 보는 개인 공간.

핵심 UI:

- 프로필 요약
- 누적 거리 / 완주 코스 / 획득 바우처
- 내 라이딩 기록 리스트
- 내 다이어리
- 내 바우처 지갑

### 3.4 Compass

목적: 설정, 언어, 데이터, 관리자 기능.

핵심 UI:

- 언어 설정
- 지도 데이터/오프라인 캐시 설정
- 로그인/로그아웃
- 개인정보/권한 안내
- 관리자 계정일 때만 Admin Console 노출

## 4. 디자인 토큰

### 4.1 색상

기존 디자인 가이드의 의도를 살리되, 앱 전체가 파란색 한 톤으로만 보이지 않도록 보조 색을 추가한다.

| 역할 | 이름 | 값 | 용도 |
|---|---|---:|---|
| Primary | K-Indigo | `#1E3A8A` | 주요 액션, 활성 탭, 브랜드 포인트 |
| Accent | Aero Blue | `#0EA5E9` | 현재 위치, 진행 상태, 보조 액션 |
| Warm Accent | Sunrise Gold | `#F59E0B` | 바우처, 리워드, 인증 완료 |
| Neutral Text | Asphalt | `#334155` | 본문 텍스트 |
| Muted Text | Stone | `#64748B` | 보조 텍스트 |
| Surface | Cloud | `#F8FAFC` | 기본 배경 |
| Card | White | `#FFFFFF` | 리스트 항목, 모달 |
| Border | Mist | `#E2E8F0` | 경계선 |
| Danger | Signal | `#DC2626` | 삭제/오류 등 필수 위험 표현에만 제한 사용 |

주의:

- 지도 자체에 초록색/빨강색이 들어갈 수 있으므로 앱 UI는 이를 과하게 반복하지 않는다.
- 위험/삭제 상태는 빨강을 완전히 금지하지 않고 기능적 의미로만 제한한다.

### 4.2 타이포그래피

Expo/RN 환경에서 안정성을 우선한다.

- 기본: 시스템 산세리프
- 추후: Pretendard, Inter 로드
- 제목: 20-24
- 섹션 제목: 16-18
- 본문: 14-16
- 보조 정보: 12-13
- 라이딩 수치: 28-36

### 4.3 Radius와 Shadow

모바일 앱의 조작감과 정돈감을 우선한다.

- 버튼: 8-12
- 카드/리스트 항목: 8
- 바텀시트 상단: 16
- 지도 위 플로팅 버튼: 원형 또는 8
- 그림자는 약하게, 경계선과 배경 대비를 우선한다.

## 7. 사용자 준비 항목

디자인 전면 개편 중 사용자가 직접 준비하거나 결정해야 할 항목이다.

- RideKorea 로고 방향: 텍스트 로고 유지 vs 심볼 제작
- 앱 아이콘 방향: 지도 핀/자전거/태극 색감 중 선택
- 브랜드 언어: `RideKorea` 그대로 갈지, 부제 추가할지
- 기본 표시 언어: 영어 우선인지 한국어 우선인지
- 네이버 지도 기본 언어 및 외국어 표기 검증
- 실제 사용할 스팟 사진/코스 이미지 샘플 제공 여부

## 8. 첫 디자인 스프린트

첫 디자인 스프린트는 화면을 예쁘게 꾸미는 것보다, 앱의 골격을 제대로 세우는 데 집중한다.

### Sprint D1 목표

RideKorea의 디자인 토큰과 4탭 구조를 코드에 반영한다.

### Sprint D1 작업

- [ ] `theme.ts`를 RideKorea 디자인 토큰으로 갱신
- [ ] 로그인 전 첫 화면을 Google Login 화면으로 구성
- [ ] 탭 구조를 Journey/Moments/My Path/Compass로 변경
- [ ] 각 탭의 기본 빈 화면 생성
- [ ] 기존 `index.tsx` 지도 화면은 Journey로 보존
- [ ] 공통 `AppButton`, `IconButton`, `StatPill` 초안 생성
- [ ] 하드코딩 색상/간격을 토큰으로 옮길 대상 표시

### Sprint D1 완료 기준

- 앱 하단 탭이 새 IA와 일치한다.
- 비로그인 첫 화면이 Google Login 화면으로 고정된다.
- 로그인 성공 후 Journey 지도 화면으로 진입한다.
- 기본 색상/간격/타이포그래피 토큰이 존재한다.
- 이후 화면 리팩토링이 디자인 시스템 위에서 진행될 수 있다.

## 9. 디자인 리스크

- 네이버 지도 자체 색상과 앱 UI 색상이 충돌할 수 있다.
- 글래스모피즘을 과하게 쓰면 지도 위 텍스트 가독성이 떨어진다.
- 라이딩 앱인데 카드가 너무 많으면 지도 중심성이 약해진다.
- 외국인 대상 앱이므로 한국어 장식 문구보다 명확한 영어 정보가 중요하다.
- 관리자 기능은 일반 사용자 IA에서 분리하지 않으면 앱이 복잡해 보인다.
