# RideKorea 핵심 구상 문서

## 1. 문서 목적

이 문서는 한국 자전거 종주를 원하는 외국인 라이더를 위한 모바일 앱 서비스의 초기 구상과 뼈대를 정리한 문서다.  
본격적인 화면 설계, 데이터베이스 설계, API 설계, 개발 태스크 분해에 들어가기 전에 서비스의 방향성을 고정하고, 다른 개발 환경이나 협업자에게도 동일한 맥락을 전달하는 것을 목적으로 한다.

## 2. 서비스 한 줄 정의

한국 자전거 종주를 원하는 외국인 라이더가 코스를 찾고, 여행을 기록하고, 자신의 추억을 지도 위에 남길 수 있는 지도 기반 자전거 여행 앱.

영문 서비스 정의:

> A map-based cycling companion for foreign riders exploring Korea's cross-country bike routes.

## 3. 핵심 타겟

### 3.1 주요 타겟

- 한국에서 자전거 국토종주를 하고 싶은 외국인
- 한국어 정보 접근이 어려운 해외 라이더
- 서울-부산, 제주도, 한강, 아라뱃길 등 대표 자전거길을 여행하고 싶은 방문객
- 텐트, 게스트하우스, 모텔, 호텔 등을 활용해 장거리 자전거 여행을 하는 사람

### 3.2 타겟의 주요 문제

- 한국 자전거길 정보가 외국어로 충분히 정리되어 있지 않음
- 국토종주 코스, 인증센터, 스탬프 문화, 교통 연결 정보를 이해하기 어려움
- 숙소, 음식점, 편의점, 자전거 수리점, 렌탈샵 정보를 자전거 여행 맥락에서 찾기 어려움
- 라이딩 중 남긴 사진과 기록을 코스/스팟 기준으로 정리하기 어려움
- 다른 라이더의 실제 경험과 스팟별 추억을 지도에서 보기 어려움

## 4. 서비스의 큰 축

### 4.1 지도 기반 자전거길 탐색

앱의 기본 화면은 지도다.  
사용자는 한국의 주요 자전거 도로, 국토종주 코스, 인증센터, 주요 스팟을 지도에서 확인할 수 있다.

초기 지도 기능:

- 한국 주요 자전거 종주 코스 표시
- 코스별 거리, 난이도, 예상 소요 기간 표시
- 인증센터, 휴식 지점, 관광 스팟 표시
- 숙박시설, 야영장, 음식점, 편의점, 자전거 수리점, 렌탈샵 표시
- 다른 사용자가 공개한 다이어리/사진 스팟 표시

### 4.2 나의 종주 기록과 스팟 다이어리

사용자는 자신의 자전거 종주 이력을 만들고, 각 스팟마다 사진과 다이어리를 남길 수 있다.

초기 기록 기능:

- 종주 코스 선택
- 나의 종주 시작/진행/완료 상태 저장
- 스팟별 사진 업로드
- 스팟별 짧은 다이어리 작성
- 공개/비공개 선택
- 공개된 기록은 지도 위의 커뮤니티 스팟으로 노출

## 5. 초기 MVP 범위

초기 버전은 전국 모든 자전거길을 다루지 않는다.  
하나의 대표 코스를 완성도 있게 제공하고, 외국인 라이더가 실제로 사용할 수 있는지 검증한다.

### 5.1 추천 MVP 코스

1. 서울-부산 국토종주 코스
2. 아라뱃길 + 한강 코스
3. 제주 환상 자전거길

초기 MVP에서는 `서울-부산 국토종주 코스`를 1순위로 검토한다.

### 5.2 MVP 필수 기능

- 회원가입/로그인
- 지도 화면
- 서울-부산 종주 코스 표시
- 주요 인증센터/스팟 표시
- 스팟 상세 정보
- 내 종주 기록 생성
- 스팟별 사진/다이어리 작성
- 공개 다이어리 지도 노출
- 영어 UI

### 5.3 MVP에서 제외할 기능

- 실시간 턴바이턴 내비게이션
- 숙소 예약
- 결제
- 실시간 채팅
- 전체 한국 자전거길 완전 커버
- 오프라인 지도
- 정교한 운동 기록 분석

위 기능들은 초기 유저 검증 후 고도화 단계에서 검토한다.

## 6. 기술 방향

### 6.1 앱 프레임워크

초기 제안:

- React Native
- Expo

선택 이유:

- iOS와 Android를 동시에 개발할 수 있음
- 앱 개발을 처음 시작할 때 진입 장벽이 상대적으로 낮음
- 로그인, 이미지 업로드, 위치 권한, 푸시 알림 등 앱 기본 기능을 빠르게 붙일 수 있음
- 초기 MVP와 프로토타입 제작에 적합함

### 6.2 백엔드 및 인증 (Open-source Stack)

- **언어 및 프레임워크**: Python (FastAPI)
- **데이터베이스**: PostgreSQL (PostGIS 공간 확장 지원)
- **선택 이유**:
  - 지도 및 위치 기반 데이터(자전거 코스, 인증센터, 스팟)를 지리적으로 조회하고 반경 검색을 위해 **PostgreSQL + PostGIS**가 최적의 오픈소스 솔루션임.
  - **FastAPI**를 사용해 성능이 우수하고 비동기(async) API 처리가 유연한 백엔드를 구축.
  - 모든 백엔드와 DB 엔진은 특정 상용 벤더에 종속되지 않고 자체 호스팅(Docker 등)이 가능한 100% 오픈소스 환경으로 구성.

### 6.2.1 로그인 및 인증 방식
- **소셜 로그인 전용**: 사용자의 번거로움을 줄이고 보안 신뢰성을 높이기 위해 **Google 및 Apple 소셜 로그인(OAuth 2.0)**만 제공.
- 백엔드(FastAPI)에서 ID 토큰(Identity Token)을 검증하고, 클라이언트(React Native 앱)에는 자체 JWT(JSON Web Token)를 발급하여 인증 상태 유지.

### 6.3 지도

- 카카오맵 (최종 선정)

선택 및 적용 방식:
- 국내의 고품질 장소 정보(POI), 자전거 전용도로 데이터가 풍부한 카카오맵을 핵심 지도 엔진으로 채택함.
- React Native/Expo 환경에서의 연동을 위해 **WebView를 통한 카카오 지도 Web API** 방식을 사용함. 웹 페이지를 통해 카카오맵을 그리고, React Native와 양방향 메시지 통신(`postMessage`)을 통해 코스 데이터 전송, 마커 클릭 감지, 다이어리 작성 팝업 등의 이벤트를 유기적으로 연동함.
- 외국인을 위해 영문 및 다국어 지원에 한계가 있는 부분은, 자체 GPX/GeoJSON 경로 안내 데이터와 핵심 포인트 이름의 영문 번역 데이터를 오버레이 레이어(Overlay Layer) 형태로 직접 지도 위에 얹어 보완함.

중요한 원칙:

자전거 경로 계산을 지도 API에 전적으로 의존하지 않는다.  
앱이 제공하는 종주 코스는 GPX/GeoJSON 형태의 자체 데이터로 관리하고, 지도 위에 오버레이한다.

## 7. 지도 API 비용/정책 검토 메모

### 7.1 카카오맵

초기 검토 기준:

- 모바일 지도 SDK 무료 사용량이 초기 서비스에는 충분한 수준
- 초과 요금 체계가 비교적 명확함
- 국내 POI와 지도 품질이 좋음

주의할 점:

- React Native 공식 SDK가 없으므로 WebView 또는 네이티브 브릿지 필요
- 외국어 표기와 외국인 사용자 경험 검증 필요

### 7.2 네이버지도

초기 검토 기준:

- 무료 제공량이 커 보임
- 국내 지도 품질과 장소 정보가 좋음

주의할 점:

- 초과 과금 구조와 사업화 조건은 추가 확인 필요
- React Native 공식 SDK가 없으므로 WebView 또는 네이티브 브릿지 필요
- 외국인 대상 UI/검색 품질 검증 필요

### 7.3 초기 결정 방향

MVP에서는 카카오맵 또는 네이버지도 중 하나를 WebView 방식으로 먼저 붙인다.  
최종 선택은 다음 기준으로 결정한다.

- 영어 사용자 경험
- 자전거 여행 POI 검색 품질
- API 비용과 무료 제공량
- React Native 연동 난이도
- 코스 오버레이, 마커, 커스텀 정보창 구현 난이도

## 8. 데이터 소스 후보

### 8.1 자전거길/종주 데이터

후보:

- 자전거행복나눔
- 공공데이터포털
- 지자체 공개 데이터
- 국가공간정보포털
- OpenStreetMap

필요 데이터:

- 종주 코스 경로
- 인증센터 위치
- 자전거길 구간 정보
- 자전거도로 유형
- 위험 구간
- 휴식 지점
- 화장실/편의시설

초기 전략:

- 공식 종주 코스는 자전거행복나눔 정보를 기준으로 정리
- 지도 표시용 경로는 GPX 또는 GeoJSON 형태로 자체 관리
- 공공데이터와 OSM 데이터는 보완 데이터로 활용
- 데이터 출처와 갱신일을 반드시 저장

### 8.2 관광/숙박/음식점 데이터

후보:

- 한국관광공사 TourAPI
- 공공데이터포털 숙박/음식점 데이터
- 카카오/네이버 장소 검색 API
- 지자체 관광 데이터

필요 데이터:

- 숙박시설
- 음식점
- 카페
- 편의점
- 관광지
- 지역 축제/행사

초기 전략:

- 숙박시설은 추후 예약 연동 가능성을 고려해 데이터 모델 설계
- 외국인 사용자를 위해 영어명, 주소, 카테고리, 위치 좌표를 우선 확보

### 8.3 야영장/캠핑 데이터

후보:

- 고캠핑 데이터
- 한국관광공사 캠핑장 데이터
- 공공데이터포털 야영장 데이터

필요 데이터:

- 캠핑장 위치

- 예약 가능 여부

초기 전략:

- 텐트 여행자를 위한 별도 필터로 제공
- 고도화 단계에서 예약 링크 또는 예약 연동 검토

### 8.4 자전거 렌탈/수리 데이터

후보:

- 카카오/네이버 장소 검색

- 지자체 자전거 대여소 데이터
- 공공자전거 데이터

필요 데이터:

- 자전거 렌탈샵
- 자전거 수리점
- 공공자전거 대여소
- 출발지 근처 렌탈 가능 지점

초기 전략:

- 서울/부산/제주 등 출발지 중심으로 먼저 정리
- 실제 외국인이 이용 가능한지, 영어 대응 가능 여부는 커뮤니티 피드백으로 보완

## 10. 사용자 경험 방향

### 10.1 첫 화면

앱을 열면 바로 지도와 대표 코스가 보여야 한다.  
일반적인 소개 페이지보다, 실제로 한국 자전거 여행을 시작할 수 있는 느낌이 중요하다.

### 10.2 외국인 친화 요소

- 영어 UI 우선
- 한국 주소와 영어 주소 병기
- 인증센터/종주 문화 설명
- 기차/버스/자전거 이동 팁
- 응급상황 표현
- 한국 자전거길 표지판 설명
- 코스별 추천 일정

### 10.3 커뮤니티 요소

공개 다이어리는 지도 위에서 볼 수 있게 한다.  
각 스팟은 단순한 장소 정보가 아니라, 실제 라이더들의 경험이 쌓이는 지점이 된다.

## 11. 고도화 방향

### 11.1 여행 지원

- 오프라인 지도
- GPX 다운로드
- 구간별 날씨
- 고도 정보
- 위험 구간 신고
- 자전거 이동 가능한 대중교통 정보

### 11.2 예약/지역 연계

- 숙박 예약 연동
- 캠핑장 예약 링크
- 지역 음식점 추천
- 지자체 추천 코스
- 지역 관광공사/지자체 홍보 페이지

### 11.3 수익 모델 후보

초기에는 고려하지 않는다.  
다만 장기적으로는 다음 모델을 검토할 수 있다.

- 지방자치단체/관광공사 홍보 제휴
- 지역 숙박시설/음식점 제휴
- 프리미엄 코스팩
- 오프라인 지도 유료 기능
- 완주 리포트/포토북
- 외국인 대상 자전거 여행 패키지 연계

### Phase 0. 기획 정리

- 핵심 서비스 정의 확정
- 타겟 사용자 정의
- MVP 범위 확정
- 지도 API 후보 비교
- 데이터 소스 후보 정리

## 13. 현재 기준 결정 사항

- **앱 명칭**: RideKorea (가칭)
- **대상 플랫폼**: iOS 및 Android (React Native + Expo)
- **백엔드 기술 스택**: 100% 오픈소스 구성
  - **API 서버**: Python (FastAPI + SQLAlchemy + Uvicorn)
  - **데이터베이스**: PostgreSQL (PostGIS 확장 기능 포함)
- **인증 방식**: Google 및 Apple 소셜 로그인 전용 (자체 이메일/비밀번호 가입 없음)
- **지도 엔진**: 카카오 지도 Web API (WebView 기반 연동)
- **경로 데이터**: 카카오 지도 경로 안내 대신, 자체 GPX/GeoJSON 파일을 지도 위에 선으로 오버레이 표시.
- **초기 MVP 코스**: 서울-부산 국토종주 코스
