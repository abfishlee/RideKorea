# RideKorea 지도 제공자(Map Provider) 의사결정 문서

> [!summary]
> **결론:** 기본 지도·검색은 **네이버(NAVER Cloud Platform Maps)**, 자전거 라우팅은 **자체 OSM 엔진(GraphHopper/BRouter)**, 지도 계층은 **제공자 비종속 추상화(`MapProvider`)**로 구성한다. **Mapbox는 오프라인·고도·코스 타일셋 보조**로, **VWorld는 국내 한정 데이터 소스**로만 사용한다. **Google Maps는 한국 내비 정식 출시 대기 후보.**
> 상태: ✅ 결정(2026-06-25) · 재검토 트리거: 구글맵 한국 턴바이턴 정식 출시 시

작성일: 2026-06-25

---

## 1. 배경

초기 기획은 카카오맵(WebView)이었으나, 라이더가 **전용 자전거길뿐 아니라 일반 도로**도 주행한다는 요구가 추가되면서 "자체 GeoJSON 오버레이"만으로는 부족해졌다. 일반 도로 수준의 정확한 도로 데이터 + 라우팅 + 외국인 대상 다국어가 동시에 필요해졌고, 이에 따라 지도 제공자를 재검토한다.

핵심 제약: **한국은 정밀지도(1:5,000) 국외 반출 규제**로 외국 지도사(구글·애플·Mapbox)의 국내 도로/POI/라우팅이 약하거나 부재하다.

### 후보별 핵심 메모

- **네이버** — 한국에서 유일하게 4개 언어(한·영·중·일)를 지도·POI·리뷰까지 지원. 국내 지도 시장 약 70% 점유로 도로·POI 데이터가 가장 촘촘. 단, 길찾기 API(Directions 5/15)는 **자동차 기준**.
- **카카오** — 소비자 앱 영문은 양호하나 개발 API 다국어 라벨은 네이버보다 약함. 외국인 타겟에서 네이버에 열세.
- **구글맵** — 2026-02-27 한국 정부가 1:5,000 지도 반출을 **조건부 승인**(19년 분쟁 종료). 향후 6개월 내 턴바이턴 출시 예상이나, 2026-04 시점까지 **여전히 미출시·정체**. 지금 의존 불가, 단 향후 강력 후보.
- **Mapbox** — 개발 편의·오프라인·지형/고도·커스텀 타일셋은 최고. 그러나 한국 기본 도로/POI 데이터가 약해 "일반 도로" 요구에선 단독 부적합.
- **VWorld(브이월드)** — 무료·국가공간정보 풍부하나 **공간정보관리법 제16조로 해외 사용 불가**, 라우팅 엔진 아님, 다국어 라벨 없음 → 앱 지도 엔진으로 부적합.

---

## 3. 핵심 난점: 자전거 라우팅

네이버·카카오·구글의 길찾기 API는 **전부 자동차/도보 기준**이다. 한국에는 **프로그래밍 가능한 자전거 전용 라우팅 API가 사실상 없다.** 자동차 경로를 자전거에 그대로 쓰면 고속도로·자동차전용도로로 안내되어 위험·위법이다.

→ 일반 도로 자전거 길찾기는 **OSM 기반 라우팅 엔진(GraphHopper/BRouter/OSRM, bike 프로파일)을 자체 운영**하는 것이 현실적 유일 해법. OSM 한국 데이터는 도심 양호, 시골 다소 성김.

---

## 4. 결정 아키텍처

지도 시장이 실시간으로 변동 중(구글 반출 승인 직후)이므로 **단일 제공자 종속을 끊는 추상화**가 가장 중요한 결정이다. RideKorea의 "경로는 자체 데이터로 관리" 철학과도 일치한다.

1. **지도 추상화 계층(`MapProvider`)** — 배경지도 렌더, 라우트 GeoJSON 그리기, 마커, 화면 맞춤, 오프라인 캐시를 제공자 비종속 인터페이스로 정의. 구현체 교체 가능.
2. **기본 렌더러/검색: 네이버** — 다국어 + 한국 도로 정확도. 비용 최적화를 위해 **Web Dynamic Map(WebView)** 방식 우선(무료 이용량 대상, §6 참고).
3. **라우팅 3단 폴백** — ① 공식 종주 코스 = 자체 GPX/GeoJSON(최우선) → ② 일반 도로 = 자체 OSM bike 엔진 → ③ 실패 시 도보 경로 근사. 자동차 경로는 자전거 안내에 사용 금지.
4. **Mapbox = 보조 특기 영역** — 음영지역 오프라인 타일, 고도 프로파일, 자체 코스 벡터 타일셋. 주력 렌더러는 하나(네이버)만 운영.
5. **VWorld = 보조 데이터 소스** — 행정경계·지적 등 국내 한정 레이어. 앱 지도 엔진으로는 미사용.
6. **Google = 대기 후보** — 정식 출시 시 추상화 계층을 통해 저비용 채택/전환 후 재비교.

## 5. 구현 매핑

- 프론트엔드 추상화: `frontend/src/map/MapProvider.ts`(인터페이스), `providers/NaverMapProvider.tsx`(기본), `providers/MapboxMapProvider.tsx`(보조), `routing/OsmBikeRoutingProvider.ts`(OSM 호출), `index.ts`(팩토리).
- 백엔드 라우팅: `backend/app/services/routing/`(base·osm_bike·orchestrator), `routers/routing.py`(`POST /routing`).
- 자전거 엔진: 별도 컨테이너로 GraphHopper(권장) 또는 BRouter를 OSM 한국 추출본으로 구동. 환경변수 `OSM_ROUTING_BASE_URL`로 연결.

## 6. 네이버 사용 비용 (요약)

- **과금 모델:** 종량(구간) 요금제. 호출/뷰 생성 건당 과금.
- **무료 이용량:** **대표 계정에 한해** Web Dynamic Map / Static Map / Geocoding / Reverse Geocoding에만 제공. **Mobile Dynamic Map(네이티브 SDK)과 Directions는 무료 이용량 대상 아님** → 첫 호출부터 과금.
- **건수 산정:** Mobile Dynamic Map = 지도 뷰 1회 생성(onCreate) 시 1건 / Web Dynamic Map = 로딩 시에만 1건(zoom·marker 등 인터랙션은 미과금).
- **비용 최적화 포인트:** RN에서 네이티브 Mobile Dynamic Map 대신 **Web Dynamic Map을 WebView로** 쓰면 무료 이용량 적용 + 기존 WebView 브리지 구조 재사용 → 비용·구조 모두 유리.
- **정확한 단가:** NCP 콘솔/공식 요금 페이지·요금계산기에서 확인(콘솔 외 안정적 공개 단가가 없고 2023.1 개정 이력 있음). 운영 전 일별/월별 이용 상한선을 콘솔에서 설정해 과금 폭주 방지.
- **이용 조건:** 네이버 기능·명칭(거리뷰/길찾기/내비게이션 등)을 임의 변경 없이 동일 표기. 한국 외 리전(US/SG/JP/DE)도 제공되어 해외 접속 라이더 대응 가능.

## 7. 네이버 최소 비용 운영 전략 (Cost-Minimization Playbook)

**원칙: "네이버엔 지도 렌더만, 검색·좌표변환·라우팅은 무료 대안으로."** 과금이 발생하는 지점을 구조적으로 최소화한다.

### 체크리스트

1. **렌더러는 Web Dynamic Map(WebView)만 사용.** 네이티브 Mobile Dynamic Map SDK는 무료 이용량이 없으므로 금지. (`NaverMapProvider`가 이미 WebView 방식)
2. **대표 계정 1개로 통일.** Web Dynamic Map·Geocoding·Reverse Geocoding 무료 이용량은 대표계정에만 적용된다.
3. **Geocoding/Reverse Geocoding 호출 제거.** POI 좌표는 공공데이터(TourAPI·자전거길 DB)에서 좌표 포함 형태로 사전 적재 → 런타임 좌표변환 호출을 없앤다. 부득이 필요하면 결과를 DB에 캐싱.
4. **장소/키워드 검색은 자체 구현.** 네이버 유료 검색 대신 우리 DB(spots)·TourAPI 기반 검색으로 대체.
5. **라우팅은 네이버 Directions 호출 0건.** 자체 OSM 엔진(GraphHopper)만 사용.
6. **지도 로딩(=과금) 횟수 줄이기.** 화면 재진입 시 WebView/지도 인스턴스를 재사용(언마운트·재생성 방지). Web Dynamic Map은 "로딩 시 1건"이므로 인스턴스 보존이 곧 비용 절감.
7. **콘솔에서 일/월 이용 상한선 설정** → 비정상 과금·폭주 차단.
8. **클라이언트 ID는 환경변수(.env) 주입** + 도메인/번들ID 제한 설정으로 키 오남용 방지(평문 하드코딩 금지).
9. **정적 이미지 화면은 Static Map 검토.** 썸네일·공유 카드 등 비상호작용 화면은 무료 이용량 대상인 Static Map 활용.
10. **사용량 모니터링.** NCP 콘솔 Usage Statistics로 월별 추세 추적, 무료 한도 근접 시 점검.

### 효과

위를 지키면 과금 트리거가 **"지도 화면 로딩" 하나로 수렴**하고, Web Dynamic Map 무료 이용량 내에서 초기·파일럿은 사실상 무료~소액으로 운영된다. 핵심 비용 변수는 "지도를 몇 번 새로 띄우느냐"이므로, 지도 인스턴스 캐싱이 가장 큰 절감 레버다.

### 비고

- 정확한 무료 이용량 수치·단가는 NCP 콘솔/요금계산기에서 확인.
- 사업화자금(관광벤처·예비창업패키지) 집행 시 지도 API·클라우드 비용은 '지급수수료' 비목으로 **협약기간 내** 집행 가능성이 높음(주관기관 PD 사전확인 필요). 협약 종료 후 운영비는 별도 지속 모델(B2G·제휴·후속펀딩)로 충당.
