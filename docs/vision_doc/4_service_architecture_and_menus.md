# 4. 서비스 아키텍처 및 메뉴 구조 (Service Architecture & Menus)

## 📱 앱 하단 탭 내비게이션 구조 (Bottom Tabs)

사용자의 여정(Before -> During -> After)을 커버하는 4개의 메인 탭으로 구성됩니다.

### 🔍 1. Explore (탐색) - *Before Ride*
사사키가 종단 루트를 검색하고 피터의 여행기를 발견하는 탭입니다.
* **추천 루트 (Feeds)**: 유저들이 공유한 기행문(루트+사진+메모)이 인스타그램 피드처럼 예쁘게 나열됩니다.
* **물류 가이드 (Logistics Tips)**: 공항에서 출발지까지 이동하는 법, 자전거 포장법 등 실무적인 팁이 게시판 형태로 제공됩니다.
* **Action**: 마음에 드는 타인의 루트를 발견하면 `[나의 경로로 가져오기]` 버튼을 누릅니다.

### 🚴‍♂️ 2. Ride (주행) - *During Ride*
가장 핵심이 되는 네이버 지도 기반의 실시간 주행 화면입니다.
* **Full-Screen Map**: 지도가 전체 화면을 덮습니다.
* **Floating Dashboard**: 상단에 반투명하게 '현재 속도, 주행 거리, 고도표'가 오버레이 됩니다.
* **Photo Pin Button**: 화면 우측 하단에 거대한 카메라 버튼. 누르면 즉시 카메라가 켜지고, 사진을 찍으면 현재 GPS 좌표에 핑(Pin)이 떨어지며 짧은 메모를 남길 수 있습니다.
* **Voucher Popup**: 특정 지역(Polygon) 진입 시, 하단에서 '지역 바우처 획득!' 카드가 애니메이션과 함께 올라옵니다.

### 📖 3. Diary (기록) - *After Ride*
주행이 끝난 후, 내가 남긴 궤적과 사진 핀들을 모아 나만의 기행문을 감상하고 편집하는 공간입니다.
* **My Journeys**: 내가 주행했던 궤적(Blue + Pink)과 위치별 사진, 메모들이 타임라인 형태로 묶여 하나의 완성된 여행기로 제공됩니다.
* **Share**: 이 다이어리를 다른 유저들에게 공개(Public)하거나 인스타그램에 내보낼 수 있습니다.

### 🎫 4. Wallet (지갑) - *Local Economy*
라이딩 중 모아둔 지역 상권 할인 쿠폰과 종주 스탬프를 관리하는 공간입니다.
* **보유 바우처**: 식당, 숙박, 수리점 쿠폰 바코드 리스트.
* **스탬프 패스포트**: 주요 거점을 지날 때마다 찍힌 디지털 종주 스탬프 여권 화면.

## ⚙️ 시스템 동작 흐름 (사사키 시나리오 아키텍처)

**(※ AI 개발 주의사항: 백그라운드 추적 및 바우처 트리거 로직을 구현할 때, 프론트엔드와 백엔드의 책임을 명확히 분리하십시오.)**

1. **(App) 주행 시작**: 앱이 `expo-location`의 백그라운드 태스크를 사용하여 GPS 추적을 시작하고, 위치 데이터를 SQLite(로컬 DB)에 5초 단위로 임시 기록합니다.
2. **(App) 사진 찰칵**: 사진을 찍으면 **Supabase Storage**로 즉시 업로드되고, 얻은 URL과 현재 GPS 좌표(Point)를 **Supabase DB** `route_spots` 테이블에 Insert 합니다.
3. **(App & DB) 바우처 트리거 (Geofencing)**: 앱이 1분 주기로 현재 위치를 Supabase에 보내면, PostGIS의 `ST_Contains(region_polygon, current_location)` 쿼리를 통해 특정 바우처 구역 진입 여부를 서버가 판단하고 프론트엔드로 알림(Push/Popup)을 트리거합니다.
4. **(App) 주행 종료**: 주행 종료 버튼을 누르면, 로컬에 모아둔 전체 GPS 좌표 배열을 GeoJSON `LineString`으로 변환하여 **Supabase DB** `routes.path_geom`에 통째로 쏟아붓습니다. (Sync)
5. **(Data Sync)**: DB에 `routes` 데이터 저장이 완료되면, 클라이언트(앱) 화면은 즉시 '기록(Diary)' 탭으로 전환되어 사용자가 방금 주행한 궤적과 사진들을 타임라인 형태로 확인할 수 있게 합니다.
