# RideKorea 고도화 로드맵 및 리팩토링 계획서

본 문서는 RideKorea 앱의 상용 서비스 수준 도약을 위해 주말 동안 진행할 대대적인 아키텍처 개편 및 기능 고도화에 대한 마스터플랜입니다.

---

## 1. 앱 아키텍처 및 화면 설계 (UI/UX)
* **주제**: 단일 화면(One-Screen) 탈피 및 모던 네비게이션 구조 도입
* **문제점**: 현재 모든 UI 컴포넌트와 지도, 버튼들이 `index.tsx` 한 곳에 집중되어 있어 코드가 복잡하고(스파게티 코드), 사용성이 매우 떨어집니다. UI 디자인이 촌스럽습니다.
* **개선방향**:
  * Expo Router의 **Bottom Tab Navigation**을 도입하여 화면을 독립적인 탭(지도/주행, 피드/다이어리, 검색, 마이페이지)으로 분리합니다.
  * TailwindCSS (NativeWind) 등을 도입하여 여백과 타이포그래피 중심의 깔끔한 모던 UI(Glassmorphism 등)를 적용합니다.
* **개선 전 선택 사항**:
  * 디자인 시스템 프레임워크 선택: `NativeWind (Tailwind)` vs `UI Kitten` vs `자체 Custom StyleSheet`
  * 하단 탭의 구체적인 메뉴 구성 확정 (예: [Riding], [Community], [Discover], [Profile])

## 2. 지도 엔진 교체 및 고도화
* **주제**: 카카오맵을 대체할 글로벌/아웃도어 특화 지도 엔진 도입
* **문제점**: 카카오맵은 영문 지원이 제한적이고, 고도(Elevation) 표시나 글로벌 POI(외국인 친화적 장소 데이터) 기능이 부족합니다.
* **개선방향**:
  * **Mapbox** 또는 **Google Maps(react-native-maps)** 로 지도 엔진을 전면 교체합니다.
  * 자전거 도로 레이어(Bicycle Layer) 활성화 및 라이딩 경로의 고도 프로필 차트를 추가합니다.
  * 숙박, 식당, 캠핑장 등 라이더를 위한 주요 인프라(POI) 마커를 영문으로 지도 위에 표시합니다.
* **개선 전 선택 사항**:
  * 지도 API 선택: 
    * `Mapbox` (디자인과 아웃도어 3D 지형에 압도적 강점, 단 초기 세팅이 조금 복잡함)
    * `Google Maps` (전 세계 장소 데이터 최강, 무난한 세팅)

## 3. 개인 커스텀 코스 및 백그라운드 GPS 트래킹
* **주제**: 나만의 경로 주행 기록 및 SNS 다이어리 연동
* **문제점**: 현재는 서버에서 내려주는 고정된 단일 종주 코스만 존재하며, 내가 실제로 달린 궤적을 기록하고 저장하는 기능이 없습니다.
* **개선방향**:
  * `expo-location`을 활용해 라이딩 시작 시 **백그라운드 위치 추적**을 실행하고 이동 궤적을 수집합니다.
  * 주행 종료 후 수집된 좌표를 GPX나 GeoJSON 형태로 백엔드에 저장하고, 이를 장소 다이어리 사진과 묶어 피드에 공유할 수 있게 합니다.
* **개선 전 선택 사항**:
  * 위치 추적 주기 설정: 배터리 소모를 고려하여 GPS 좌표 수집 간격(예: 5초 or 10미터 단위) 결정
  * 오프라인 주행 지원 여부: 인터넷이 끊긴 산간 지역에서도 내부 DB(SQLite/AsyncStorage)에 좌표를 임시 저장할지 여부

## 4. 코스 데이터 다양화 및 확장
* **주제**: 단일 더미 코스에서 실제 전국 자전거길 및 테마 코스로 확장
* **문제점**: 종주 코스 데이터가 하드코딩된 단일 샘플뿐이어서 서비스로서의 볼륨이 부족합니다.
* **개선방향**:
  * 백엔드(PostGIS)에 행정안전부 자전거길 공공데이터(GPX)를 파싱하여 전국 코스를 밀어넣습니다.
  * 코스를 '4대강 종주', '해안 힐링', '도심 속 라이딩' 등의 테마로 그룹화하는 API를 구축합니다.
* **개선 전 선택 사항**:
  * 외부 경로 데이터 출처 확정: 공공데이터포털 API 연동 vs Strava/Komoot API 크롤링 및 제휴 vs 관리자 페이지에서 GPX 파일 직접 업로드 기능 구현

## 5. 다국어(i18n) 시스템 및 스플래시/초기 로딩 완벽 적용
* **주제**: 글로벌 서비스 퀄리티 확보를 위한 언어팩 및 UX 개선
* **문제점**: 언어 변경 버튼이 작동하지 않으며, 앱 실행 시 GPS가 잡히기 전 한국 전체 지도가 먼저 렌더링되어 어색합니다.
* **개선방향**:
  * `i18next` 라이브러리를 도입해 텍스트를 JSON 파일로 분리하고, 영어/일본어/중국어/한국어를 완벽 지원합니다.
  * 앱 실행 시 스플래시 스크린을 띄운 상태로 위치 권한을 먼저 체크하고, 현재 위치 좌표를 확보한 뒤에야 지도를 줌인 애니메이션과 함께 렌더링합니다.
* **개선 전 선택 사항**:
  * 번역 리소스 관리 방식 결정: 로컬 JSON 파일로 관리 vs 백엔드 API를 통해 번역본 실시간 업데이트
